title: "Skills"

---

A skill is an approved procedure that tells Mothy how your organisation wants a
recurring task done: a report format, a triage rule, a house convention. Skills
let you shape how Mothy works without any technical change, and the rule is that
Mothy can draft a skill but only a person makes it real.

## Using skills

Mothy keeps a short index of the available skills always to hand, and reads the
full detail of a skill only when it is relevant to what you have asked. A skill is
treated as guidance and can never override Mothy's core rules.

## Suggesting a change

Mothy can propose a new skill, or a change to one, based on what it has learned
from working with you. It never applies the change itself. It stages a proposal
that an administrator reviews and approves or rejects on the
[Proposals and approvals](/mothy-docs/proposals-and-approvals/) screen. Each
approved change is recorded, so the history of a skill is always clear.

## Keeping the library tidy

Skills that have not been used for a while are marked as out of date and then
retired automatically, so the library does not fill with stale procedures. Nothing
is ever truly deleted; a retired skill can be brought back by an administrator.
