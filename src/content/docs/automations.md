title: "Automations"
description: "Approved jobs that Mothy runs on a schedule, in a fresh audited session."
---

An automation is an approved job that Mothy runs on a schedule: a prompt, with any
attached skills, run on its own and the result saved. Typical uses are a morning
summary of new documents, a weekly task digest, or a calendar-aware briefing.

## Creating one

An administrator sets up an automation, or approves one that Mothy has drafted, on
the Automations screen, and it runs on the schedule you choose. Because it is a
standing instruction that repeats, it goes through the same approval as a skill
before it can run.

## How a run works

Each run happens in a fresh, self-contained session, uses only the model it was
approved for, and can only read, never change anything. Every run is recorded,
with its result and whether it succeeded, and you can see the history and pause or
resume an automation at any time. If anything about an automation changes, the
change is recorded, and nothing runs silently.
